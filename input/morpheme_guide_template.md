# 형태소 분석 지침 (Morpheme Analysis Guide)

이 파일은 `python main.py morpheme --guide <path>` 커맨드가 LLM 프롬프트에
통째로 주입하는 지침서 **템플릿** 입니다. 내용을 수정하면 분해 품질과 처리
속도가 직접 변합니다. 배치 처리 규칙 섹션도 함께 유지·보수하세요.

사용법:

```bash
cp input/morpheme_guide_template.md input/my_morpheme_guide.md
# my_morpheme_guide.md 를 프로젝트 도메인에 맞게 수정 후
python main.py morpheme --input attrs.txt --guide input/my_morpheme_guide.md
```

`.gitignore` 가 `input/*_template.*` 만 커밋 허용하므로 사용자 버전은 자동으로
gitignore 처리됩니다.

## 역할

당신은 반도체 제조 및 공급망 시스템의 데이터 표준화 전문가입니다.
주어진 속성명을 다음 규칙에 따라 정확히 형태소 분해하여 단어 목록으로
반환합니다.

## 기본 원칙

1. **언어 경계 자동 분리** — 한글 ↔ 영문 ↔ 숫자 전환점에서 자른다.
   공백/기호 (`-`, `_`, `/`, `.`) 는 필수가 아니다.
   - `BACKEND지역값` → `["BACKEND", "지역", "값"]`
   - `WAFER수량2024` → `["WAFER", "수량", "2024"]`

2. **구분자 처리** — `-`, `_`, `/`, `.`, 공백은 경계로만 사용하고 결과에는
   포함하지 않는다.
   - `LOT_ID_LIST` → `["LOT", "ID", "LIST"]`
   - `eqp/chamber-no` → `["eqp", "chamber", "no"]`

3. **CamelCase / PascalCase 분리** — 대문자 전환점에서 자른다. 연속 대문자는
   약어로 묶는다.
   - `waferLotId` → `["wafer", "Lot", "Id"]`
   - `FABRunRate` → `["FAB", "Run", "Rate"]`

4. **업계 표준 약어 보존** — 반도체/공급망 도메인 약어는 단일 토큰으로 유지.
   예시: `FAB`, `WIP`, `LOT`, `RMA`, `BOM`, `PPID`, `MES`, `APC`, `SPC`,
   `EDA`, `RFID`, `CIM`, `SECS`, `GEM`.

5. **숫자 suffix 규칙** — 숫자 + 영문자 1글자 이하 suffix (`Q1`, `H2`) 는
   결합해 한 토큰. 그 외는 분리.
   - `Lot2024Q1` → `["Lot", "2024", "Q1"]`
   - `Code2Name` → `["Code", "2", "Name"]`

6. **한글 단어 보존** — 한국어 형태소 분석기 수준으로 어간/어미를 분해하지
   않는다. 업무 단어 단위로만 유지한다.
   - `설비상태코드` → `["설비", "상태", "코드"]` (O)
   - `설비상태코드` → `["설", "비", "상", "태", "코", "드"]` (X)
   - `설비가동률현황` → `["설비", "가동률", "현황"]` (O — 단일 개념 "가동률"
     은 분해 금지)

## 출력 형식

**반드시 유효한 JSON 배열**만 응답하세요. 설명 / 코드펜스 / 부가 텍스트 금지.

```json
[
  {"attr": "BACKEND지역값", "tokens": ["BACKEND", "지역", "값"], "confidence": 0.98},
  {"attr": "LOT_ID",        "tokens": ["LOT", "ID"],            "confidence": 1.00}
]
```

필드 정의:

- `attr`: 입력 속성명을 그대로 반환 (수정 금지)
- `tokens`: 분해 결과 배열. 순서는 원본 등장 순서 유지
- `confidence`: 0.0 ~ 1.0. 모호한 약어 / 드문 케이스는 0.7 미만으로 낮춰
  수동 검토 큐에 들어가게 한다

## Few-shot 예시 (모델 학습용 — 순서 유지)

아래는 분해 규칙의 적용 예시입니다. LLM 은 이 패턴을 기준으로 동일한 JSON
배열 구조로 응답해야 합니다. 각 예시는 하나 이상의 기본 원칙을 커버합니다.

### 예시 1 — 언어 경계 분리 (한↔영↔숫)
- 입력: `BACKEND지역값2024`
- 출력: `{"attr": "BACKEND지역값2024", "tokens": ["BACKEND", "지역", "값", "2024"], "confidence": 0.98}`
- 적용 규칙: 원칙 1

### 예시 2 — 구분자 기반 분리
- 입력: `WAFER_LOT_ID`
- 출력: `{"attr": "WAFER_LOT_ID", "tokens": ["WAFER", "LOT", "ID"], "confidence": 1.00}`
- 적용 규칙: 원칙 2

### 예시 3 — CamelCase 분리 + 연속 대문자 약어 처리
- 입력: `FABRunRate`
- 출력: `{"attr": "FABRunRate", "tokens": ["FAB", "Run", "Rate"], "confidence": 0.95}`
- 적용 규칙: 원칙 3 + 원칙 4

### 예시 4 — 한글 단어 보존 (형태소 과분해 금지)
- 입력: `설비가동률현황`
- 출력: `{"attr": "설비가동률현황", "tokens": ["설비", "가동률", "현황"], "confidence": 0.92}`
- 적용 규칙: 원칙 6

### 예시 5 — 업계 표준 약어 보존
- 입력: `PPID관리번호`
- 출력: `{"attr": "PPID관리번호", "tokens": ["PPID", "관리", "번호"], "confidence": 1.00}`
- 적용 규칙: 원칙 4

### 예시 6 — 숫자 + 단일 영문자 suffix 결합
- 입력: `Lot2024Q1Start`
- 출력: `{"attr": "Lot2024Q1Start", "tokens": ["Lot", "2024", "Q1", "Start"], "confidence": 0.85}`
- 적용 규칙: 원칙 5

### 예시 7 — 엣지 케이스 (신뢰도 낮음, 단일글자 플래그)
- 입력: `CHK_Y_N`
- 출력: `{"attr": "CHK_Y_N", "tokens": ["CHK", "Y", "N"], "confidence": 0.70}`
- 적용 규칙: 단일글자 플래그 (`Y`, `N`) 는 그대로 토큰 유지, 신뢰도는 0.7

## 배치 처리 규칙 (시스템 운영용)

형태소분석 커맨드는 아래 규칙으로 속성명을 묶어서 LLM 에 전달합니다. 이
섹션은 배치 크기 튜닝의 근거 자료이며 `morpheme_analyzer.py` 의 상수와
일치해야 합니다.

- **권장 배치 크기: 30** (속성 평균 길이 10~20자 기준)
- **자동 조정 공식**: `max(10, min(50, 1200 // avg_attr_len))`
  - 속성 평균 30자 이상 (긴 복합어 많음) → 20 부근으로 내려감
  - 속성 평균 8자 이하 (짧은 약어) → 50까지 올라감
- **JSON 실패 시 재시도**: 배치 크기를 절반으로 축소해 최대 2단계 재시도
  (30 → 15 → 7). 3회째 실패 시 해당 범위는 파싱 실패로 기록
- **temperature**: 0.0 ~ 0.1 (결정적 응답 필요)
- **timeout**: 배치당 120초
- **병렬도**: 기본 1. `--parallel N` 으로 증가 가능하나 사내 게이트웨이
  rate limit 확인 후 사용

## 배치 크기 trade-off 참고표

| 배치 | 입력 토큰 | 출력 토큰 | 1콜 체감시간 (중형 로컬 LLM) | 리스크 |
|------|-----------|-----------|------------------------------|--------|
| 10   | ~500      | ~400      | 3~6초                        | 호출 오버헤드 지배 |
| 30   | ~800      | ~1,200    | 8~15초                       | 최적 (JSON 안정) |
| 50   | ~1,200    | ~2,000    | 15~25초                      | 긴 출력에서 JSON 깨짐 증가 |
| 100  | ~2,400    | ~4,000    | 30~50초                      | JSON 중간 잘림 빈발 |

## 처리 속도 추정 (2만 건 기준)

| LLM 환경 | 스루풋 | 콜당 (배치30) | 순차 총시간 | 4병렬 |
|---------|--------|---------------|-------------|-------|
| Ollama 중급 CPU | 30 tok/s | ~40초 | 7.4시간 | 1.9시간 |
| vLLM 중급 GPU   | 80 tok/s | ~15초 | 2.8시간 | 42분 |
| 사내 게이트웨이 | 200+ tok/s | ~6초 | 67분 | 17분 |

## 품질 체크 (리포트 단계에서 자동 검증)

- `len(tokens) == 0` → 파싱 실패 플래그 (빨강 하이라이트)
- `len(tokens) > 12` → 단어12까지만 저장 + 비고에 "13번째 이후 N개 생략"
  기록 (파랑 하이라이트)
- `confidence < 0.7` 이고 `> 0.0` → 저신뢰도 (노랑 하이라이트, 수동 검토
  큐)

## Few-shot 선정 원칙 (지침 유지관리용)

- **규칙 커버리지**: 원칙 1 ~ 원칙 6 각각 최소 1개 예시 포함 (현재 7개 중
  1~6 은 각 원칙, 7 은 엣지 케이스)
- **다양성**:
  - 길이: 짧음 (3자) / 중간 (10자) / 긺 (20자+) 모두 포함
  - 언어 조합: 영문전용 / 한글전용 / 한영혼합 / 한영숫 3중혼합 모두 포함
  - 도메인: 반도체 약어 (FAB / PPID) + 일반 CRUD 용어 (LIST / REG / UPD)
- **중복 제거**: 같은 규칙만 반복되는 예시는 1개로 통합
- **순서**: 쉬운 것 → 복잡한 것 (모델이 규칙 난이도를 학습하도록)
- **권장 개수**: 5 ~ 8 (초과 시 recency bias, 미만 시 규칙 누락)
- **업데이트 주기**: 실제 저신뢰도 사례가 반복 발견되면 해당 케이스를
  예시 8 ~ 10 으로 추가
